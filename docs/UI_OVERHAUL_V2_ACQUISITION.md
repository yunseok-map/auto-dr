# auto-dr UI 전면 개편 스펙 — "ACQUISITION" (신호 포착 콘솔)

> 버전: V2 (완전 신규 컴셉 · 과감한 재구성)
> 전제: **기능·데이터 흐름·API·상태 로직은 100% 보존.** 프레젠테이션 레이어만 재설계한다.
> 기존 brass/teal "계측기" 정체성은 폐기하고, 아래의 새 정체성으로 교체한다.
> 이 문서가 구현의 단일 기준(single source of truth)이다.

---

## 0. 왜 바꾸는가 (진단)

현재 UI는 디자인 토큰·색·타이포 수준은 높지만, `detail` 화면이 **동일한 둥근 사각형 카드(.section)를 세로로 무한히 쌓은 "카드 스택"** 이다. 결과적으로:

- 모든 정보가 같은 위계·같은 모양으로 평평하게 나열된다 (위계 부재).
- "계측기" 컨셉인데 정작 **계기판처럼 보이지 않는다** — 그냥 폼+카드 대시보드.
- 제품의 핵심 서사(= *반복하며 최고 품질로 수렴한다*)가 화면에서 드러나지 않는다.

→ 개편의 목표: **제품의 서사를 화면 구조 자체로 만든다.** 한 화면을 보면 "아, 이건 노이즈에서 신호를 잠가내는 기계구나"가 즉시 읽혀야 한다.

## 1. 컴셉: ACQUISITION (신호 포착)

**은유:** 전파 망원경/장노출 천문 관측. 한 번의 관측(integration)은 노이즈투성이지만, **여러 번 누적(stack)하면 약한 신호가 노이즈 위로 떠오른다(lock-on).** 

이 은유가 제품과 1:1로 맞는다:

| 제품 개념 | 관측 은유 |
|---|---|
| 한 번의 리뷰·개선 반복(iteration) | 한 번의 노출/적분(integration / pass) |
| 회차 점수 0~100 | 신호 강도(signal strength) |
| 베스트 점수 채택 | 신호 잠금(LOCK) |
| 점수 정체/수렴 | 잡음 바닥 도달(noise floor) — 더 적분해도 안 오름 |
| 모델 승급(haiku→sonnet→opus) | 감도 증폭(gain stage up) |
| 지적사항 대장(findings) | 채널/스펙트럼 라인이 하나씩 해소됨 |
| 비용 상한 | 관측 예산(observing budget) |

**기억에 남을 한 가지(signature):** 회차별 점수를 **신호 포착 트레이스**로 시각화한 대형 패널 — 노이즈 바닥, 누적되며 잠기는 골드 LOCK 데이텀, 펄스하는 라이브 노드. 그리고 그 옆의 거대한 **신호 강도 리드아웃**이 수렴 시 "LOCK" 상태로 점등한다. 보드 전체에서 **딱 이 패널 하나에만** 과감함을 쓴다. 나머지는 전부 조용하고 규율 있게.

**Anti-AI 체크:** 크림+세리프+테라코타 아님. "순흑 배경 + 단일 애시드 그린/버밀리언" 함정을 피하기 위해 (a) 배경은 순흑이 아니라 **청자색 야간(blue-violet night)**, (b) 단일 애시드 대신 **역할이 분명한 다색 기능 팔레트 + 차가운 바탕에 단 하나의 따뜻한 골드(LOCK 전용)** 로 간다. 이모지 아이콘 금지(인라인 스트로크 SVG 유지). 균일 카드 그리드 금지.

## 2. 팔레트 (역할별 named hex)

차가운 야간 바탕 + 차가운 시그널 애저 + **유일한 따뜻한 골드(잠금 전용)**. 온도 대비가 정체성의 핵심.

### 바탕/표면 (cool blue-violet night)
- `--void`     `#0A0E1A` — 앱 최하단 바탕 (청자색 야간, 순흑·청록 아님)
- `--deep`     `#0F1525` — 패널 베이스
- `--panel`    `#151D32` — 카드/레일/올라온 표면
- `--panel-2`  `#1C2740` — 인셋/트랙/호버
- `--line`     `#263352` — 헤어라인/보더
- `--line-lit` `#33446B` — 강조 보더, 입력 테두리

### 잉크 (cool whites)
- `--ink`      `#EAEFFB` — 본문/리드아웃 (대비 ≈ 16:1 on void)
- `--ink-dim`  `#AEB9D6` — 보조 텍스트
- `--faint`    `#6E7CA0` — 3차 텍스트, 축 라벨, 틱
- `--faint-2`  `#48557A` — 가장 흐린 틱/그리드

### 시그널 — 1차 애센트 (cool azure; 라이브/링크/액티브)
- `--signal`     `#4FD6FF` — THE 시그널: 라이브 트레이스, 액티브, 링크
- `--signal-hi`  `#7FE4FF`
- `--signal-press` `#34B8E0`
- `--signal-ink` `#04202B` — 애저 채움 위 어두운 글자
- `--signal-wash` `rgba(79,214,255,0.12)`
- `--signal-line` `rgba(79,214,255,0.42)`

### LOCK — 유일한 따뜻한 골드 (베스트/수렴/잠금 "순간"에만)
- `--lock`      `#FFD27A` — 베스트 리드아웃, LOCK 데이텀, 신호 잠금
- `--lock-hi`   `#FFDF9C`
- `--lock-ink`  `#2C1E03`
- `--lock-wash` `rgba(255,210,122,0.13)`
- `--lock-line` `rgba(255,210,122,0.40)`
> 규율: 골드는 **포착된 신호/잠금**에만 쓴다. 버튼·링크·일반 강조에 절대 쓰지 않는다. 차가운 바탕 위 단 하나의 따뜻한 점 → 시선이 자동으로 "결과"에 꽂힌다.

### 기능색 (상태)
- `--ok`     `#5CE6A6` (positive: 채택/해결/개선) · `--ok-ink` `#04231A` · `--ok-wash` `rgba(92,230,166,0.13)`
- `--warn`   `#F2B45C` (caution: 일시정지/주의) · `--warn-ink` `#2A1C02` · `--warn-wash` `rgba(242,180,92,0.13)`
- `--bad`    `#FF6B6B` (error/regression: 항상 글리프+라벨 동반) · `--bad-ink` `#2A0606` · `--bad-wash` `rgba(255,107,107,0.14)`

### 깊이/분위기
- 단 하나의 매우 낮은 알파 성운 그라데이션을 바탕에 고정(과하지 않게):
  `radial-gradient(900px 460px at 88% -12%, rgba(79,214,255,0.06), transparent 62%)` + 아주 옅은 두 번째 보라 누출 `radial-gradient(620px 380px at 6% 8%, rgba(124,108,255,0.045), transparent 60%)`.

색만으로 의미를 전달하지 않는다: error/warn/ok는 글리프(원-X / 삼각 / 체크) + 위치 + 텍스트 라벨을 항상 동반.

## 3. 타이포그래피 (텔레메트리 우선)

정체성의 큰 전환점: **모노스페이스를 데이터 공동 주역으로 승격.** 천문/텔레메트리 콘솔의 핵심은 정렬된 수치 데이터다.

- **디스플레이(대형 리드아웃·브랜드):** `Bricolage Grotesque`(이미 self-host 중인 woff2 재사용). 단, 사용처를 바꾼다 — 거대한 신호 리드아웃과 마스트헤드 타이틀에만, 더 크고 더 타이트하게(letter-spacing -0.03em).
- **텔레메트리/데이터(신규 주역):** 시스템 모노 스택 `ui-monospace, "Cascadia Code", "JetBrains Mono", Consolas`. 점수·좌표·타임스탬프·축 라벨·통계·로그·테이블 수치 전부 모노. `font-variant-numeric: tabular-nums` 강제. 이게 "콘솔" 느낌을 만든다.
- **본문/UI(한글 포함):** `Pretendard, -apple-system, "Apple SD Gothic Neo", "Malgun Gothic", "Segoe UI", system-ui`.

새 폰트 바이너리 추가 없음(오프라인/무의존 원칙 유지).

### 타입 스케일
- `--fs-readout` `clamp(48px, 8vw, 96px)` — 신호 강도 리드아웃 (디스플레이, 단 1곳)
- `--fs-h1` `clamp(22px, 2vw+15px, 30px)` — 마스트헤드 타이틀
- `--fs-h2` `15px` — 섹션 에이브로우 (모노, uppercase, tracking .12em)
- `--fs-lg` `clamp(16px, .6vw+14px, 18px)`
- `--fs-base` `14px`
- `--fs-sm` `12.5px`
- `--fs-xs` `11px`
- `--fs-2xs` `10px` (틱/유닛 라벨, 모노)

섹션 제목은 더 이상 "제목"처럼 굵게 쓰지 않는다. **모노 대문자 에이브로우 + 좌측 채널 인덱스 틱**(예: `CH·02 ⟩ 신호 포착`) 형태의 텔레메트리 라벨로.

## 4. 레이아웃 & 위계 (과감한 재구성)

### 4.1 스페이싱/라디우스/엘리베이션
- 8pt 스케일 유지: 4/8/12/16/20/24/32/48/64.
- 라디우스: 콘솔 느낌을 위해 **전반적으로 더 샤프하게** — `--r-xs 3px / --r 6px / --r-lg 10px`. 패널 모서리는 작게, 칩은 거의 각지게.
- 엘리베이션: 컬러 글로우 금지. 야간 바탕에 맞춘 순수 블랙 섀도 1/2/3단.
- **표면 차등화가 핵심(= 카드 스택 탈피):** 최소 3종 표면을 시각적으로 구분 —
  1. **올라온 패널**(panel, 보더+섀도) — 시그니처 포착 패널.
  2. **레일/그루핑**(보더 없이 배경 톤 차이 + 헤어라인 구분선) — 부가 데이터.
  3. **인셋/리세스**(panel-2, inset shadow) — 트랙·뷰어·로그처럼 "안으로 들어간" 영역.
  모든 블록이 같은 카드면 안 된다.

│  NEW ACQUISI- │                                              │
│  TION         │              DETAIL (콘솔)                    │
│  ┌─ 파일/URL ─┐│                                              │
│  │   드롭존   ││                                              │
│  └───────────┘│                                              │
│  초점 ▾        │                                              │
│  ▸ 고급(예산)  │                                              │
│  [ 포착 시작 ] │                                              │
│  ─────────────│                                              │

- 좌측 채널 틱·모노 에이브로우로 텔레메트리 톤 통일.

### 4.3 사이드바 — 타깃 카드 (재설계)
각 런 = 하나의 "포착 대상". 카드 안에 **인라인 미니 신호 스파크라인**(최근 점수 추이)과 상태 점등을 넣는다.
```
┌────────────────────────────────────┐
│ 보고서_v3.docx                      │  title
│ DOC · ● 진행중        ╭╮  ╭─╮       │  kind·status  + 미니 스파크라인
│                       ╯ ╰─╯  ▮84   │  + best score (모노, 골드면 LOCK)
└────────────────────────────────────┘
```
- active 타깃: 좌측 골드 채널 바 + 살짝 밝은 배경.
- 점수는 모노 tabular. 베스트가 갱신되어 "잠긴" 상태면 골드.

### 4.4 DETAIL — 포착 콘솔 (핵심 재구성)
카드 스택을 폐기하고 4개 "구역(zone)"으로:

```
╔══ ZONE 1 · 마스트헤드 ════════════════════════════════════════╗
  보고서_v3.docx                         [⏸ 일시정지][■ 중단]
  ● 진행중 · DOC · C:\...\보고서.docx · 좌표(소스)
╚══════════════════════════════════════════════════════════════╝

╔══ ZONE 2 · ACQUISITION 패널 (시그니처, 풀블리드, 올라온 패널) ══╗
 ┌─ 신호 포착 트레이스 ───────────────────┐  ┌─ 리드아웃 ────────┐
 │ 100 ┄┄┄┄┄┄┄┄┄┄ LOCK 데이텀(골드 점선) │  │ SIGNAL STRENGTH   │
 │     ╱●╲   ╱●─live(펄스)              │  │                   │
 │  ╱─╯   ╲╱                            │  │     84            │ ← 골드 디스플레이
 │ ·noise floor·······                  │  │     ▲ LOCK #5     │   (수렴 시 LOCK 점등)
 │  #1  #2  #3  #4  #5                   │  │  +12 첫 패스 대비  │
 └──────────────────────────────────────┘  └───────────────────┘
 ── 텔레메트리 스트립(패널 하단, 인셋) ───────────────────────────
  PASS 5/6 · GAIN sonnet→opus · COST $0.18 / cap $0.50 · 12.3s avg
╚══════════════════════════════════════════════════════════════╝

╔══ ZONE 3 · 작업 영역 (2단; 레일 차등화) ══════════════════════╗
 ┌─ 메인: CHANNELS · 지적사항 대장 ──────┐ ┌─ 레일 ──────────────┐
 │ ▰▰▰▰▰▱▱  해결 5/8 (62%)              │ │ DIMENSIONS          │
 │ ⟩ #3 [high] 입력 검증 누락           │ │ 가독성   ▮▮▮▮▮▯ 82  │
 │ ⟩ #5 [med]  네이밍 불일치            │ │ 보안     ▮▮▮▮▯▯ 70  │
 │ ▸ 해결된 항목(3)                      │ │ 구조     ▮▮▮▮▮▮ 91  │
 │                                       │ │ ─────────────────── │
 │                                       │ │ 남은 포인트         │
 │                                       │ │ · …                 │
 └───────────────────────────────────────┘ └─────────────────────┘
╚══════════════════════════════════════════════════════════════╝

╔══ ZONE 4 · 기록 & 산출물 ════════════════════════════════════╗
 INTEGRATION LOG (패스별 표; 점수 셀에 미니 막대)
 #  STR  채택  GAIN(모델)  소요   근거
 1  62   ·     haiku       8.1s   …
 5  84   ✓     opus        12s    …
 ── READOUT (탭: 리뷰 / 개선본 / diff) — 인셋 뷰어 ──
 ── TELEMETRY FEED (로그 콘솔) ──
╚══════════════════════════════════════════════════════════════╝
```

위계 규칙:
1. ZONE 2(포착 패널)가 시각적으로 압도. 가장 큰 면적·유일한 골드·유일한 큰 디스플레이 숫자·유일한 모션.
2. ZONE 3 메인(대장)이 그다음. 레일(dimensions/이슈)은 한 단계 조용히.
3. ZONE 4는 가장 조용한 "아카이브" 톤(인셋/리세스).
4. 카드 보더의 균일 반복 금지 — 패널/레일/인셋 3종을 섞어 리듬을 만든다.

## 5. 컴포넌트 사양 (요점)

- **마스트헤드 바(헤더):** sticky, blur, 좌측 브랜드(▣ 모노그램 + `auto·dr` + `ACQUISITION CONSOLE` 에이브로우), 우측 알림 토글 + LIVE 인디케이터(SSE 연결 시 시그널 애저 점등 펄스).
- **포착 패널:** 단일 올라온 패널. 좌(트레이스 canvas) + 우(리드아웃, 좌측 헤어라인 구분). 하단 풀폭 텔레메트리 스트립(인셋, 모노, ` · ` 구분자). 라이브일 때만 상단 1px 스캔 셀 셔머.
- **신호 리드아웃:** `--fs-readout` 디스플레이 골드 숫자. 아래 작은 모노 메타(`LOCK #5` / `진행 #6`), 델타(`+12 첫 패스 대비`, 양수면 ok 글리프). 수렴/완료 상태에서 "LOCK" 배지 점등.
- **타깃 카드(사이드바):** title + (kind 칩 · status 칩) + 미니 스파크라인(canvas 또는 인라인 SVG, 최근 ≤12 점수) + best score(모노). active 시 골드 채널 바.
- **채널/대장:** 진행 막대(해결 비율) → "신호 채널 해소" 톤. open 항목은 시그널 애저 좌측 바, resolved는 ok 그린 좌측 바+취소선+딤. severity 칩 유지(high=bad, med=warn, low=signal).
- **DIMENSIONS(레일):** 수평 신호 막대(트랙=인셋, 채움=점수색 그라데이션, 값=모노). 막대를 더 얇게(6px), 텔레메트리 게이지 느낌.
- **INTEGRATION LOG(표):** 컬럼 헤더 모노 대문자. 점수 셀에 미니 인라인 막대. 채택 행은 좌측 ok 틱. 선택 행은 골드 좌측 틱. hover 리세스.
- **READOUT 뷰어:** 탭(리뷰/개선본/diff). 인셋 패널. 마크다운/코드/diff 렌더는 기존 로직 유지, 색만 신규 토큰.
- **TELEMETRY FEED(로그):** 모노 콘솔. info/warn/error 레벨 글리프 유지(색만 교체). 타임스탬프 faint.
- **알림 배너(자동 일시정지):** warn 톤. "신호 정체 — 관측 일시중단" 카피. 액션 버튼(확인하고 재개 / 여기서 중단).
- **토스트:** 좌측 레벨 바 + 글리프. 위치/동작 유지.
- **버튼:**
  - 1차(포착 시작): 시그널 애저 채움, 어두운 잉크. hover 살짝 상승.
  - 콘솔 버튼(일시정지/재개/중단): 고스트 + 역할색 보더(warn/ok/bad).
  - 링크: 시그널 애저, 밑줄 없음, hover 밝아짐.
  - **골드는 버튼에 쓰지 않는다.**

## 6. 모션 (목적 있는 것만; reduced-motion 존중)

- 포착 트레이스 좌→우 드로우인(기존 유지, 신규 톤). 560ms ease-out.
- 라이브 노드 펄스(running일 때만): 골드 헤일로가 2.4s로 호흡.
- 리드아웃 점수 카운트업(런 전환·갱신 시 짧게).
- LIVE 인디케이터 펄스(SSE 연결 시).
- 패널/존 진입 `rise`(8px, 380ms) 1회.
- 라이브 포착 패널 상단 1px 스캔 셔머(running 한정, 매우 은은).
- 그 외 정지. 산만한 효과 금지.
- `@media (prefers-reduced-motion: reduce)` 전역 차단 유지.

## 7. 카피 (디자인 재료로 다룬다)

영문 텔레메트리 키워드(에이브로우/유닛)는 콘솔 톤을 위해 소량 사용하되, **실사용 문구는 한국어·능동·평서체.**
- 섹션 에이브로우: `SIGNAL` / `CHANNELS` / `INTEGRATION LOG` / `READOUT` / `TELEMETRY` / `DIMENSIONS` 같은 짧은 라벨 + 한국어 부제.
- 버튼: "포착 시작"(리뷰 시작), "일시정지/재개/중단" 유지.
- 빈 상태: 행동 유도 ("대상을 선택하거나 새 포착을 시작하세요").
- 에러: 무엇이/왜/어떻게 — "파일을 선택하세요", "URL 또는 경로를 입력하세요" 유지·정제.
- 필러 카피 금지("워크플로우를 강화하세요" 류 절대 금지).
- 상태 라벨(한국어) 유지: 진행중/일시정지/완료/정체로 중단/비용 상한 도달 등.

## 8. 구현 단계 (이 순서로)

> 건드리는 파일은 **프레젠테이션 레이어만**: `src/dashboard/public/styles.css`, `src/dashboard/public/index.html`, `src/dashboard/public/app.js`. 서버/오케스트레이터/타입 등 비즈니스 로직은 손대지 않는다. JS의 모든 핸들러·fetch·id 훅·SSE·폼 전송 동작을 보존한다.

- **P1 토큰 기반:** styles.css 상단을 새 팔레트/타이포/스페이싱/라디우스/섀도/모션 토큰으로 교체. 차트가 읽는 토큰명(`--signal`,`--lock`,`--ok`,`--line`,`--faint`,`--void`)을 정의. 레거시 별칭(--brass→--lock 등)은 두지 않고 **호출부를 새 토큰으로 갱신**(app.js 차트 포함).
- **P2 셸:** 마스트헤드, 레이아웃, 사이드바(새 리뷰 폼 + 타깃 목록), 빈 상태. index.html 마크업 소폭 조정(브랜드 에이브로우, 라벨), CSS 재작성.
- **P3 포착 패널(시그니처):** renderDetail의 ZONE1+ZONE2 마크업 재작성 + CSS. 차트 `_drawChart`를 신호 포착 스타일로 갱신(노이즈 바닥 라벨, 골드 LOCK 데이텀, 라이브 펄스 노드, 신규 색).
- **P4 작업 영역:** ZONE3(대장 메인 + dimensions/이슈 레일) 마크업·CSS.
- **P5 기록/산출물:** ZONE4(integration log 표 + readout 뷰어 + telemetry feed) 마크업·CSS.
- **P6 사이드바 타깃 카드 + 미니 스파크라인** 렌더 추가.
- **P7 카피·라벨·상태칩·아이콘** 정리. 반응형(모바일 드로어/스택) 재확인. reduced-motion 확인.
- 각 단계 후 동작 무결성 확인(핸들러/탭/제어/SSE/폼).

## 9. 자기비평 게이트 (스크린별 1~10, 모두 ≥8 & 동작무결성=10까지 반복)

1. 고유성 — §1 AI 디폴트를 피했는가
2. 위계 — 시선이 포착 패널 → 대장 → 아카이브 순으로 가는가
3. 타이포 — 모노 텔레메트리/디스플레이/본문 역할 분리가 의도적인가
4. 간격·리듬 — 스케일에서 파생됐는가, 표면 차등이 사는가
5. 모션 — 목적적인가, reduced-motion 존중하는가
6. 카피 — 구체·능동·사용자 관점, 필러 없음
7. 동작 무결성 — 모든 기능이 그대로 동작(=10 필수)
8. 접근성 — 포커스 링, 대비 AA, 시맨틱, 모바일, 색만으로 의미 전달 안 함
