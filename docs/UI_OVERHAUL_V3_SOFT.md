# auto-dr UI 전면 개편 스펙 — "DRAFTS" (소프트 모던 프로덕트)

> 버전: V3 (밝고·둥글고·친근한 고급 앱 / 살아 움직이는 "분석 중" 상태)
> 전제: **기능·데이터·API·상태 로직 100% 보존.** 프레젠테이션만 재설계.
> V2(다크 텔레메트리 콘솔)는 폐기. 다크+네온+모노+차트 = AI 대시보드 클리셰라 전부 버린다.
> 이 문서가 구현의 단일 기준이다.

---

## 0. 왜 또 바꾸나 (방향)

V2는 결국 "다크 배경 + 네온 + 모노 + 큰 차트"라 한눈에 **AI가 만든 대시보드**처럼 보였다. 사용자가 원하는 건:

1. **누가 봐도 공들인, 독특한 특색** — 흔한 다크 대시보드가 아니라 "잘 만든 소비자 제품" 느낌.
2. **반응형** — 데스크탑/태블릿/모바일 모두 자연스럽게.
3. **살아 움직이는 "분석 중"** — 돌아갈 때 "지금 분석 중이다, 다듬는 중이다"가 명확히 느껴지는 라이브 상태. 이게 최우선.

→ 컨셉: **"DRAFTS" — 내 문서를 한 장(초안)씩 다듬어 주는 다정한 스튜디오.** 기계 계기판이 아니라, 옆에서 같이 글을 고쳐주는 친근하고 프리미엄한 도구.

---

## 1. 무드 / 안티-클리셰

- 밝은 따뜻한 배경(순백 아님, 옅은 라일락-웜 톤) + **둥근 카드** + 부드러운 그림자 + 생동감 있는 마이크로 모션.
- 시그니처 컬러 = **친근한 바이올렛/그레이프**(테라코타·애시드그린·버밀리언 같은 AI 디폴트 회피). 에너지는 따뜻한 **피치** 한 점으로.
- 타이포는 모노 텔레메트리 금지. 큰 숫자/헤드라인은 디스플레이체, 나머지는 깔끔한 산스(한글 Pretendard). 숫자는 tabular.
- 큰 기술적 라인차트 제거 → **초안 타임라인(스텝퍼)** + **점수 링**으로 친근하게.
- 이모지 아이콘 금지(인라인 스트로크 SVG). 균일 카드 그리드 금지(히어로 크게, 나머지 작게).

## 3. 타이포

- **디스플레이(큰 숫자·헤드라인):** `Bricolage Grotesque`(self-host 재사용), weight 600–700, letter-spacing -0.02em.
- **본문/UI(한글 포함):** `Pretendard, -apple-system, "Apple SD Gothic Neo", "Malgun Gothic", "Segoe UI", system-ui`.
- 숫자 `font-variant-numeric: tabular-nums`. 모노는 코드/diff 뷰어에만.
- 스케일: `--fs-hero clamp(44px,7vw,80px)`(점수), `--fs-h1 clamp(22px,2vw+15px,30px)`, `--fs-lg 17px`, `--fs-base 14.5px`, `--fs-sm 13px`, `--fs-xs 11.5px`.
- 라인하이트 넉넉히(본문 1.6). 친근한 호흡.

---

## 4. 라운드/간격/모션

- 라디우스 크게: `--r-sm 10px / --r 16px / --r-lg 22px / --r-pill 999px`. 칩·버튼은 pill.
- 8pt 스페이싱. 카드 패딩 넉넉(20–26px).
- **모션이 주인공(= "분석 중" 살아있음):**
  - **분석 중 오브(orb):** 점수/패스 숫자를 감싼 원 둘레로 `--live-grad` 링이 회전(2.2s). 안쪽은 흰 코어 + 현재 패스 번호. 은은한 호흡 스케일.
  - **펄스 도트:** `◉○○ → ○◉○ → ○○◉` 1.2s 순환 — "분석 중…" 옆.
  - **라이브 진행바:** 부정형(indeterminate) 셔머가 좌우로 흐름.
  - **카드 호버:** `translateY(-2px)` + 그림자 성장.
  - **점수 카운트업**, **초안 스텝퍼 그려지기**, **체크 항목 팝인**.
  - 완료 순간 살짝 **피치 글로우** 1회(축하).
  - 전부 `prefers-reduced-motion`에서 정지.
- 라이브 상태에서 사이드바의 해당 분석 행도 작은 펄스로 "돌고 있음"을 표시.

## 5. 레이아웃 (반응형 · 위계)

### 5.1 셸
```
┌───────────────────────────────────────────────┐
│ ◖drafts        분석 중 2건 ●          [알림] [☺]│  ← 밝은 탑바, 라이브 칩
├──────────────┬────────────────────────────────┤
│ + 새 분석     │   DETAIL                        │
│ ┌ 드롭존 ┐    │                                 │
│ 분석 목록     │                                 │
│ ◔ 보고서.docx │                                 │
│ ◕ 코드.js     │                                 │
└──────────────┴────────────────────────────────┘
```

### 5.2 분석 목록 카드(사이드바)
원형 진행 링 + 제목 + 상태. 라이브면 링이 회전 + "분석 중".
```
( ◔ 67% )  보고서_v3.docx
           분석 중 · 3번째 다듬는 중   ← 라이브 펄스
```

### 5.3 DETAIL — 히어로 중심(상태로 분기)
**A) 분석 중 / 대기 (running·pending) — 라이브 히어로(주인공):**
```
╭───────────────────────────────────────────────╮
│                                       [⏸][■]   │
│   ╭─live ring─╮   분석 중…  ◉○○                │
│   │    03     │   지금 3번째 초안을 다듬고 있어요 │
│   ╰───────────╯   모델 ◗ Opus · 12.3s째          │
│   ▁▂▃ 라이브 진행 셔머 ▃▂▁                        │
╰───────────────────────────────────────────────╯
```
**B) 완료/중단 (done·stopped_*) — 결과 히어로:**
```
╭───────────────────────────────────────────────╮
│                                  [다시][⤓ 다운로드]│
│   ( ◕ 88 )   가장 좋았던 건 3번째 초안이에요      │
│   점수 링     +12점 좋아졌어요 · 총 3번 다듬음     │
│   ●──●──◉   초안 스텝퍼                          │
│   [ 개선본 보기 ]                                │
╰───────────────────────────────────────────────╯
```
**C) 일시정지 (paused) — 차분한 안내 + 재개 CTA.**

### 5.4 아래 섹션(부드러운 카드, 크기 차등)
- **다듬은 항목(findings):** 만족스러운 체크리스트 + 진행 퍼센트(원형/막대). 열린 항목 = 바이올렛 점, 해결 = 민트 체크.
- **평가(dimensions):** 부드러운 가로 막대(트랙 둥근 pill, 채움 점수색).
- **초안 기록(passes):** 가로 타임라인/리스트 — 각 초안 점수·채택·모델·근거.
- **미리보기:** 탭(리뷰/개선본/원본대비) — 둥근 인셋 뷰어.
- **진행 기록(log):** 친근한 라이트 리스트(타임스탬프 + 메시지). 라이브면 마지막 줄 강조.

히어로가 압도적으로 크고, 나머지는 작고 조용하게. 카드 균일 반복 금지.

### 5.5 빈 상태
다정한 일러스트(문서 + 반짝) + "분석할 문서를 올려주세요" + 행동 유도.

## 6. 카피 (친근·능동·격려)

- 분석 중: "분석 중…", "지금 N번째 초안을 다듬고 있어요", "모델 ◗ Opus · 12.3s째".
- 결과: "가장 좋았던 건 N번째 초안이에요", "+12점 좋아졌어요", "총 N번 다듬었어요".
- 항목: "다듬은 항목 13 / 15", "아직 손볼 곳 2가지".
- 버튼: "개선본 보기", "원본 형식으로 받기", "일시정지", "재개", "그만두기".
- 빈/에러: "분석할 문서를 올려주세요", "파일을 선택하세요", "주소나 경로를 입력하세요".
- 필러·기계어 금지("워크플로우" 류 금지). 상태칩 한국어 유지.

## 7. 구현 단계 (프레젠테이션 파일만)

> `src/dashboard/public/{styles.css,index.html,app.js}` 만 수정. 서버/오케스트레이터/타입/핸들러/SSE/폼/ id 훅 동작 보존.

- **P1 토큰/베이스:** styles.css 라이트 토큰·타이포·라디우스·그림자·모션 전면 교체.
- **P2 셸:** 밝은 탑바(브랜드 `drafts`, 라이브 "분석 중 n건" 칩), 사이드바(새 분석 + 진행 링 목록), 빈 상태.
- **P3 라이브 히어로(핵심):** renderDetail 상태 분기(working/paused/done) — 분석 중 오브+펄스도트+라이브 셔머, 결과 점수링+초안 스텝퍼+CTA.
- **P4 섹션:** findings 체크리스트, dimensions 막대, passes 타임라인, viewer, log(라이트).
- **P5 모션:** orb 회전·호흡, 펄스도트, 셔머, 카운트업, 호버, 완료 글로우. reduced-motion 차단.
- **P6 반응형:** 모바일 드로어, 히어로 스택, 막대/타임라인 가로스크롤. 라이트 테마 대비 AA 확인.
- 큰 캔버스 라인차트는 제거(초안 타임라인이 대체).

## 8. 자기비평 게이트 (스크린별 1~10, 모두 ≥8 & 동작무결성=10)

1. 고유성(다크 대시보드/AI 디폴트 탈피) 2. 위계(히어로→항목→기록) 3. 타이포 4. 간격·라운드 리듬 5. **라이브감(분석 중이 살아있는가)** 6. 카피(친근·능동) 7. 동작 무결성(=10) 8. 접근성(라이트 대비 AA·포커스·모바일·색만 의존 금지).

핵심 합격선: **"분석 중" 상태를 봤을 때 한눈에 '아 지금 돌아가고 있구나'가 느껴질 것.**
