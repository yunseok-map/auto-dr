# auto-dr 성능·품질 고도화 (코드 근거 기반)

> 기능이 아니라 **내부 동작의 정확성/비용/렌더링 효율**을 끌어올리는 작업.
> 코드 전체를 읽고 도출. 상태: ⬜ 대기 / 🔧 작업중 / ✅ 완료 / ⏸ 보류
> 각 항목은 파일:라인 근거를 가진다. 한 번에 다 하지 않고 위에서부터 단계별로.

## 🔧 엔진 — 품질/비용/속도

### E1. ✅ 채택 판정 히스테리시스 + judge 앵커 (품질 안정성 ⭐최우선)
- **문제**: `runner.ts:430` `kept = result.score >= prevBest` — 단일 judge(`judge.ts`, sonnet 단발) 점수의 ±2~3점 노이즈로 좋은 회차가 폐기되고 `noImprove` 누적 → 불필요한 승급/plateau 종료. `minDelta`는 plateau 카운터에만 쓰여 채택과 비대칭.
- **보완**:
  - 채택에 히스테리시스: `kept = prevBest == null || result.score >= prevBest - ACCEPT_EPS` (작은 ε).
  - judge에 직전 베스트(점수+짧은 컨텍스트)를 **앵커**로 전달 → 매번 0에서 재채점하며 생기는 드리프트 제거.
- 파일: `runner.ts`(채택), `judge.ts`/`prompts.ts`(앵커 인자).

### E2. ⬜ 대용량 judge 폴백 개선 (판정 일관성)
- **문제**: `runner.ts:389` `canJudge = length <= LARGE_FIRST_CHARS`. 큰 문서는 독립 채점을 포기하고 자가 점수(인플레)를 채택 기준으로 사용 → 작은/큰 런의 채택 척도가 달라짐.
- **보완**: 변경 섹션만 채점(diff-judge) 또는 차원 요약 채점으로 대용량에서도 독립 채점 유지.
- 파일: `runner.ts`, `judge.ts`.

### E3. ✅ 실패한 edit `find` 를 다음 회차에 피드백 (실사용 품질 ⭐1순위)
- **문제**: `runner.ts:769` `applyEdits` 는 유일 매칭만 적용. 모델 인용이 살짝 틀리면 edit 이 조용히 실패 → finding 영구 미해결 → `detectStuck` 반복 감지 → 자동정지/승급. **실패가 다음 프롬프트에 전달되지 않아** 모델이 같은 실수를 반복.
- **보완**:
  - 이번 회차 미적용 `find` 목록을 state 에 보관하고 다음 `buildPrompt` 에 "이 find 들은 본문에 없었다 — 정확히 다시 인용하라" 로 주입.
  - (선택) 라인 앵커/짧은 고유 부분문자열 폴백 매칭 한 단계 추가.
- 파일: `runner.ts`, `prompts.ts`(`PromptContext.failedFinds`), `types.ts`.

### E4. ⏸ 캐시 친화적 프롬프트 순서 — 보류(ROI 낮음)
- **재검토 결과**: `callAnthropic`(`providers.ts:155`)은 프롬프트 **전체를 단일 캐시 블록**으로 표시. artifact 가 회차마다 바뀌므로 그 블록은 회차 간 절대 재일치하지 않아 **단순 재배치는 효과 0**.
- 진짜 효과를 보려면 [안정 프리픽스 블록 + cache_control] / [휘발성 블록]으로 **분리**해야 하는데, 이는 6개 호출부(claudeAgent/providers/panel/judge/chunked) 시그니처를 바꾸고 **기본 경로인 `claude -p` 구독에는 적용 불가**(stdin 으로는 블록 제어 불가). 프롬프트 재배치는 품질 회귀 위험도 있음.
- → **보류**. API provider 사용 비중이 커지면 분리형 캐싱으로 재검토.

### E5. ✅ 조건부 judge (비용/지연 절감)
- **문제**: 모든 회차가 improve 1콜 + judge 1콜(2배). judge 는 채택 게이팅용인데 폐기될 회차도 채점.
- **보완**: 자가 점수가 베스트 미만이면 judge 생략(어차피 폐기), 개선을 주장할 때만 judge. 마무리 패스는 항상 judge.
- 파일: `runner.ts`.

### E6. ⏸ 승급 타겟 정밀화 (judge 최약 차원 기반)
- 길이 휴리스틱+무조건 +1단계 대신, judge 최약 차원을 보고 필요한 티어로 점프. (E1 이후 데이터 생기면 검토)
- 파일: `routing.ts`, `runner.ts`.

## 🖥 UI/UX — 확실한 개선

### U1. ✅ 증분 SSE 업데이트 (체감 ⭐1순위)
- **문제**: `app.js:1726` `es.addEventListener('change', () => loadRuns())` — 변경 1건마다 전체 런 재로딩 + 상세 통째 재렌더 → 깜빡임, 스크롤·탭·선택회차 유실, 뷰어 재fetch.
- **보완**: 바뀐 런만 패치, 스크롤/선택 상태 보존(없으면 최소한 상세 재렌더 시 상태 보존 + 디바운스).
- 파일: `dashboard/server.ts`(이벤트에 id), `app.js`.

### U2. ✅ 죽은/테마 불일치 차트 코드 제거
- **문제**: `drawChart/_drawChart`(canvas)가 호출처 없는 죽은 코드 + 현재 라이트 테마에 없는 토큰(`--signal`/`--void`/`--lock`)을 사용. 실제 차트는 SVG `scoreChartHtml`.
- **보완**: canvas 함수·모듈 변수 제거.
- **추가로 발견**: `sparkline()` 함수도 **호출처가 전혀 없는 죽은 코드**였고(미정의 토큰 `--signal`/`--lock` 사용), 헤드리스 검증에서 DOM 에 실제로 렌더되지 않음을 확인 → 함수 자체를 제거. 결과적으로 app.js 의 미정의 CSS 토큰 0.
- 파일: `app.js`.

### U3. ✅ 마크다운 렌더러 강화
- **문제**: `app.js:1577` `renderMarkdown` 은 h1~h3·ul·bold·code 만. 표·번호목록·인용·링크·중첩 없음 → 주 콘텐츠(작업노트/개선본) 깨짐.
- **보완**: 표·ol·링크·blockquote 추가.
- 파일: `app.js`.

### U4. ⏸ diff 가상화 + 대용량 처리
