# 대시보드 UI/UX 강화 계획

상세 화면(런 디테일) 중심 4개 축. 상태: ⬜ 대기 / 🔧 작업중 / ✅ 완료

---

## A. 정보 위계 정리 ✅
문제: 상세 화면이 카드가 세로로 길게 쌓여 스캔성이 낮음(히어로→비용→대장→평가→기록→토큰→미리보기→로그).

개선:
- 히어로 바로 아래 **핵심 지표 스트립**(PASS·MODEL·COST·AVG) 노출 → 한눈에 상태 파악(`telemetryStrip` 활용).
- **주 콘텐츠(미리보기)를 위로** 끌어올리고, **부가 정보(초안 기록·토큰·진행 기록)는 접이식(`<details>`)** 으로 — 진행 중이면 펼침, 종료면 접힘 기본.
- 순서: 헤더 → 히어로 → 비용게이지 → 지표스트립 → 점수추이 → 대장 → (평가·손볼곳) → 미리보기 → [접이식] 초안기록·토큰·진행기록.

## B. 점수/진행 시각화 ✅
문제: 점수 추이가 사이드바 스파크라인·히어로 스텝퍼뿐, 상세에 인터랙티브 차트 없음(미사용 canvas 코드 존재).

개선:
- **인터랙티브 SVG 라인차트**(`scoreChartHtml`): 회차별 점수 선 + 점(채택=채움/폐기=링/베스트=강조), **호버 툴팁**(회차·점수·채택/베스트·모델), 합격선(있으면) 점선.
- 미사용 canvas(`drawChart`)는 대체.

## C. 대장 ↔ 개선본 연결 ✅
문제: 지적(finding)을 클릭해도 개선본에서 위치를 못 찾음.

개선:
- 열린/해결 항목 **클릭 → 개선본 탭으로 전환 + 지적 제목 속 인용구('…'/"…")를 본문에서 찾아 하이라이트·스크롤**.
- 인용구가 없으면 개선본 탭으로 전환만(컨텍스트 제공). 항목에 커서/호버 어포던스.

## D. 마이크로 인터랙션 톤 정리 ✅
문제: 전환·호버·포커스 모션이 제각각.

개선:
- 공통 트랜지션 토큰(`--t-fast`/`--t-med`) 정리, 카드 **호버 살짝 들림**, 버튼 **누름 피드백**, 탭 **밑줄 슬라이드**, `:focus-visible` 일관 링, 토스트/배너 등장 모션 통일.
- `prefers-reduced-motion` 존중(기존 전역 가드 유지).
