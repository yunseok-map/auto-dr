## ⚡ 성능 고도화

| # | 항목 | 임팩트 | 상태 | 메모 |
|---|---|---|---|---|
| P1 | **스트리밍 출력** | ★★★ | ✅ | `live.ts` 인메모리 버스 + callNemotron `stream:true` + SSE `live` 이벤트. content/reasoning 모두 실시간 전송 |
| P2 | **렌즈에 빠른 모델 분리** | ★★★ | ✅ | nemotron 렌즈·검증은 `nemotron:fast`(추론 OFF, max_tokens 8192) → 10초 수준. 에디터는 추론 ON 유지. 렌즈 단계 ~3분→~15초 |
| P3 | **단계 축소 옵션** | ★★ | ⬜ | nemotron은 verify 패스 생략/병합 선택지. 렌즈+에디터만으로 충분한 경우 시간 절약 |
| P4 | **프롬프트 캐싱 확대** | ★★ | ⬜ | 현재 Anthropic만 `cache_control` 사용. 루브릭·참고기준 등 고정 프리픽스를 provider별로 캐싱 |
| P5 | **대형 1회차 분할/diff화** | ★★ | ⬜ | 큰 HTML/SVG 1회차가 전체본을 JSON에 담아 느리고 깨지기 쉬움 → 청크/부분출력 |
| P6 | **ETA 정교화** | ★ | ⬜ | 입력 크기 + provider 프로파일 기반 예측(현재는 과거 회차 평균만) |
| P7 | **resilience 일반화** | ★ | ⬜ | nemotron에 넣은 백오프 재시도를 다른 provider에도 적용 |
