# auto-dr — 자율 리뷰·개선 루프

로컬 문서 / 웹사이트 URL / 로컬 코드를 입력하면, **사람 개입 없이** AI가 스스로

1. 문제점·개선 추천점을 찾아 **지적사항 대장**에 쌓고,
2. 그 지적을 반영한 **개선본**을 만들고,
3. 자기가 만든 개선본을 **루브릭으로 자가 채점**해서,
4. 점수가 오르면 계속, 떨어지거나 정체되면 멈추고 **베스트 버전을 보존**합니다.

매 반복은 새로운 헤드리스 호출이며, 직전 전체 대화 대신 **압축된 `state.md` + 현재 베스트 산출물 + 열린 지적사항**만 넘겨 토큰을 아끼면서 작업을 이어갑니다. 산출물은 전부 로컬 `runs/`에 저장되고, 진행 상황은 실시간 로컬 웹 대시보드로 모두 볼 수 있습니다.

기본 백엔드는 **Claude Code CLI(`claude -p`, 구독 사용)** 이며, 원하면 **Anthropic / OpenAI / Gemini API 키**를 넣어 직접 호출할 수도 있습니다.

## 요구사항
- Node.js 18+ (검증: v22)
- 기본 백엔드(`cli`)를 쓰려면 `claude` CLI 로그인 완료 (Claude Code)
- (선택) API 직접 호출을 쓰려면 해당 provider의 API 키

## 설치

```powershell
npm install
```

## 사용법

### 방법 A — 대시보드에서 직접 (권장, CLI 불필요)
```powershell
npm run dashboard -- --open
```
브라우저에서 왼쪽 **"➕ 새 리뷰 시작"** 패널로:
1. `파일 업로드`(드래그앤드롭/클릭) 또는 `URL / 경로` 입력
2. **리뷰 초점** 선택 (종합 / 보안 / 가독성 / 성능 / 문서 명료성 / SEO·마케팅 등)
3. (선택) 고급 옵션에서 종류·모델·비용 상한·patience·최대 반복·평가기준 등 지정
4. **리뷰 시작** → 백그라운드로 자율 루프가 돌고, 진행·점수·비용·토큰·리뷰 결과가 실시간으로 표시됩니다.

산출물은 매 반복마다 `runs/<id>/iterations/iter-NNN/` 아래에 쌓이고, UI의 "산출물 보기"에서
작업 노트(마크다운 렌더) · 개선본 · 원본 대비 diff · 변경 내역을 탭으로 볼 수 있습니다.

### 방법 B — CLI로 바로 실행 (+ 대시보드 동시 기동)
```powershell
# 로컬 문서 (md/txt/pdf/docx/pptx 등)
npm run review -- "samples/sample.md" --open

# 웹사이트
npm run review -- "https://example.com"

# 로컬 코드 (파일 또는 폴더)

```
실행하면 대시보드가 함께 떠서(기본 http://localhost:4517) 진행 상황을 실시간으로 보여주고,
루프가 끝나도 대시보드는 계속 유지됩니다(Ctrl+C로 종료).

## 주요 옵션 (`review`)
| 옵션 | 설명 | 기본 |
| --- | --- | --- |
| `-k, --kind` | 입력 종류 강제 (`document`/`url`/`code`) | 자동 감지 |
| `-p, --port` | 대시보드 포트 | 4517 |
| `--patience <n>` | 비개선 연속 허용 횟수 → 초과 시 중단 | 2 |
| `--min-delta <n>` | 개선으로 인정할 최소 점수 상승폭 | 0.5 |
| `--max-iter <n>` | 안전 상한 반복 수(폭주 방지 백스톱) | 5 |
| `--model <m>` | claude 모델 고정(`opus`/`sonnet`/`haiku`) — 미지정 시 `auto`(자동 라우팅) | auto |
| `--no-dashboard` | 대시보드 없이 실행(끝나면 종료) | — |
| `--open` | 브라우저 자동 열기 | — |

대시보드의 고급 옵션에는 CLI에 없는 항목이 더 있습니다:
- **동일문제 반복감지**(`recurWindow`, 기본 3)
- **비용 상한**(`maxCostPerIterUsd`, `maxTotalCostUsd` 기본 $0.50)
- **모델별 시도 횟수**(`maxAttemptsPerModel`, 기본 2 → 자동 승급 속도 제어)
- **provider 선택**(cli / anthropic / openai / gemini) 및 provider 모델
- **다각도 리뷰(panel) 모드** · **맞춤 평가기준(rubric)** · **참고자료 첨부** · **변경 내역 생성** · **A/B 비교**

## 동작 방식 — 점진적(incremental) 누적 리뷰
매 회차 전체를 처음부터 다시 검사하지 않습니다.
- **1회차**: 대상을 전체 검토해 발견한 **모든 지적사항을 대장(`findings.md`)에 누적**하고, 개선본 전체를 작성합니다.
- **2회차 이후**: 대장의 **열린 항목을 해결**하고, **이전에 놓친 새 문제만** 추가합니다(전체 재검사 X).
  - 개선본 전체를 다시 출력하지 않고 **바뀌는 부분만 `edits`(부분 수정 패치)** 로 받아 적용 → 출력 토큰 절약.
  - 큰 문서는 열린 지적과 **관련된 섹션만 발췌**해 입력으로 보냅니다 → 입력 토큰 절약.
- 점수가 베스트 이상이면 채택(대장 갱신), 미만이면 그 회차는 폐기하고 **항상 베스트에서 재시도**합니다.

## 자동 모델 라우팅 + 자동 승급
모델을 지정하지 않으면(`auto`, 기본) 입력 분량·종류·초점을 보고 **모델 사다리**를 정합니다(LLM 호출 없는 코드 휴리스틱 → 비용 0).
- 짧은 코드·심층(보안/정합성/버그/성능) 초점 → **opus 직행**
- 중간 분량 → **sonnet → opus**
- 대용량(2만 자 초과) → **haiku → sonnet → opus** 계단식

진행이 막히거나(같은 지적 반복), 개선이 정체되거나, 한 모델에서 `maxAttemptsPerModel`회를 소진하면 **더 강한 모델로 한 단계 자동 승급**해 재시도합니다. 모델을 직접 지정하면 그 모델로 고정되며 승급은 일어나지 않습니다.

## 비용·반복 상한 (폭주 방지)
- `maxTotalCostUsd` (기본 **$0.50**): 누적 비용이 한도를 넘으면 다음 호출 전에 종료(`stopped_cost`). 대시보드에서 변경·해제 가능.
- `maxCostPerIterUsd`: 회차당 비용 상한(선택). 실제 호출에는 회차 상한과 **총상한 잔여분 중 작은 값**이 적용됩니다.
- `maxIterations` (기본 5): 안전 상한 반복 수. 실제 종료는 보통 완료/정체/비용상한이 먼저 겁니다.

## 종료 규칙
- **열린 항목이 모두 해결되고 새로 발견된 문제도 없음** → 완료
- (맞춤 평가기준 사용 시) 종합 점수가 **합격선(`passThreshold`) 도달** → 완료
- `done=true` (에이전트가 마무리 선언) → 완료
- `patience`회 연속 개선 정체/점수 하락 → 자동 종료(토큰 절약)
- 같은 항목이 `recurWindow`회 연속 안 풀림 → 더 강한 모델이 남았으면 승급, 없으면 **자동 일시정지**(검토 요청 알림)
- 총비용 상한 도달 → 종료 / `max-iter` 도달 → 안전 중단

## 일시정지 / 중단 (토큰 절약)
대시보드 상세 화면의 **⏸ 일시정지 / ▶ 재개 / ⏹ 중단** 버튼으로 런을 직접 제어합니다.
일시정지/중단은 **다음 호출 직전에 적용**되므로(진행 중인 반복은 끝까지 마침),
일시정지 상태에서는 **토큰을 전혀 사용하지 않습니다**. 중단은 진행 중인 호출까지 즉시 abort 합니다.

## 동일 문제 반복 자동 일시정지 + 알림
검토 이후에도 **같은 문제가 최근 `recurWindow`(기본 3)회 연속** 등장하고 **더 승급할 모델이 없으면**,
루프가 **스스로 일시정지**하고 다음을 수행합니다:
- 상세 화면 상단에 **강조 배너**로 반복된 문제를 표시
- 토스트 + (허용 시) **데스크톱 알림** + 비프음으로 다음 메시지를 띄움:
  > "불필요한 토큰 사용을 막기 위해 일시정지 됐습니다. 기존 내용 확인 부탁드립니다."

헤더의 **🔔 알림** 버튼으로 데스크톱 알림을 켤 수 있습니다. 내용 확인 후 **재개** 또는 **중단**을 선택합니다.

## 고급 기능

### 다각도 리뷰 패널 (panel 모드)
한 번의 호출 대신 **여러 렌즈(정합성·구조·표현·사실성 등)로 병렬 리뷰 → 중복 제거 → 진위 검증 → 에디터가 반영**하는 다단계 패스입니다. 렌즈 리뷰·검증은 저렴한 모델(haiku)로, 실제 개선만 라우팅 모델로 수행해 품질을 높이면서 비용을 관리합니다. (2회차 이후는 렌즈 1개·검증 생략의 경량 패스)

### 맞춤 평가기준 (rubric)
대시보드에서 **템플릿(문서/코드/이력서/마케팅)** 을 골라 차원·가중치·합격선을 편집합니다. 종합 점수는 각 차원 점수의 **가중평균**으로 시스템이 다시 계산하고, 합격선에 도달하면 완료 처리합니다.

### 참고자료 첨부 (references)
"반드시 지켜야 할 기준" 문서를 함께 올리면, 시작 시 **체크리스트로 1회 요약**(`refs/`)해 **매 패스에 주입**합니다. 기준을 위반하는 부분은 지적·교정 대상이 됩니다.

### 변경 내역 (emitChanges)
켜면 매 부분 수정에 **이유·관련 지적 번호**를 함께 받아 `best/changes.md`로 누적하고, 원본이 오피스 문서면 `changes.<형식>`으로도 내보냅니다.

### A/B 비교
같은 입력을 **두 설정(A/B)** 으로 순차 실행(`/api/compare`, 같은 groupId)해 결과를 나란히 비교합니다.

### 멀티 프로바이더 (Anthropic / OpenAI / Gemini)
대시보드 설정에서 API 키를 넣으면 해당 provider로 **직접 호출**합니다(자동 라우팅·승급 없이 지정 모델 고정). 키는 `data/providers.json`에 **로컬로만** 저장되며 `run.json`/설정에는 절대 들어가지 않고, API로는 "어떤 키가 설정됐는지"만 노출합니다.

### 오피스 문서 입·출력
- **입력**: `.pdf`(pdf-parse) · `.docx`(mammoth) · `.pptx`(jszip) 텍스트 추출.
- **출력**: 원본이 오피스 문서면 개선본을 **같은 형식(.docx/.pptx/.pdf)으로 재생성**(한글 폰트 임베드)해 다운로드. 원본 레이아웃 복원이 아니라 개선 내용을 담은 깔끔한 새 문서입니다.

## 산출물 구조
```
runs/<런ID>/
  input.<ext>            원본 스냅샷
  input.meta.json        입력 메타
  findings.md            지적사항 대장(누적 체크리스트 — 열린/해결 항목)
  state.md               압축된 진행 상태(다음 호출에 주입)
  run.json               전체 상태/점수/대장/비용/토큰/로그 (대시보드 데이터 소스)
  refs/                  첨부 참고자료 추출본(ref-N.md) — 첨부 시
  iterations/iter-NNN/
    review.md            그 회차 작업 노트
    improved.<ext>       그 회차 개선본
    score.json           점수·차원·근거·해결/새발견·열린항목
  current/               가장 최근 개선본
  best/                  점수 최고 스냅샷: review.md + improved.<ext>
    improved.<docx|pptx|pdf>   원본 형식 재생성본(오피스 입력 시)
    changes.md / changes.<fmt> 변경 내역(emitChanges 사용 시)
```

## 구조
- `src/cli.ts` — CLI (commander): `review` / `dashboard` / `list`
- `src/config.ts` — 기본 설정·경로 상수(`DEFAULT_CONFIG`)
- `src/types.ts` — 공용 타입(`RunConfig`/`RunState`/`IterationResult` 등)
- `src/input/adapters.ts` — 문서/URL/코드 입력 어댑터 + 오피스 텍스트 추출
- `src/output/export.ts` — 개선본/변경내역을 오피스 형식(docx/pptx/pdf)으로 재생성
- `src/orchestrator/`
  - `runner.ts` — 자율 반복 루프(채택·승급·종료 판정)
  - `claudeAgent.ts` — 헤드리스 호출/JSON 파싱(`runAgent`/`runPlain`)
  - `providers.ts` — Anthropic/OpenAI/Gemini API 직접 호출 + 키 관리
  - `routing.ts` — 입력 트리아지 → 모델 사다리/자동 승급
  - `panel.ts` — 다각도 리뷰 패널(렌즈·검증·에디터)
  - `prompts.ts` — 프롬프트/렌즈/섹션 스코프
  - `rubrics.ts` — 평가기준 템플릿·정규화·가중평균
  - `controls.ts` — 일시정지/재개/중단 신호 및 자식 프로세스 abort
  - `state.ts` — 런 영속화(run.json/findings.md/state.md/산출물)
- `src/dashboard/` — Express + SSE 서버 및 정적 프론트(외부 CDN 의존 없음)
