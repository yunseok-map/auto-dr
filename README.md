# auto-dr — 자율 리뷰·개선 루프

로컬 문서 / 웹사이트 URL / 로컬 코드를 입력하면, **사람 개입 없이** Claude Code가 스스로
1. 문제점·개선 추천점을 찾아 **리뷰 리포트**를 쓰고,
2. 그 리뷰를 반영한 **개선본**을 만들고,
3. 자기가 만든 개선본을 **루브릭으로 자가 채점**해서,
4. 점수가 오르면 계속, 떨어지면(또는 정체되면) 멈추고 **베스트 버전을 보존**합니다.

매 반복은 새로운 `claude -p` 헤드리스 호출이며, 직전 전체 대화 대신 **압축된 `state.md` + 현재 베스트 산출물**만 넘겨 토큰 사용을 최소화하면서 작업을 이어갑니다. 산출물은 전부 로컬 `runs/`에 저장되고, 진행 상황은 실시간 로컬 웹 대시보드로 전부 볼 수 있습니다.

## 요구사항
- Node.js 18+ (검증: v22)
- `claude` CLI 로그인 완료 (Claude Code)

## 설치
```powershell
npm install
```

## 사용법

### 방법 A — 대시보드에서 직접 (권장, CLI 불필요)
```powershell
npm run dashboard -- --open
```
브라우저에서 왼쪽 **"➕ 새 리뷰 시작"** 패널로:
1. `파일 업로드`(드래그앤드롭/클릭) 또는 `URL / 경로` 입력
2. **리뷰 초점** 선택 (종합 / 보안 / 가독성 / 성능 / 문서 명료성 / SEO·마케팅)
3. (선택) 고급 옵션에서 종류·patience·최대 반복·모델 지정
4. **리뷰 시작** → 백그라운드로 자율 루프가 돌고, 진행 상황·점수·리뷰 결과가 실시간으로 화면에 표시됩니다.

리뷰는 매 반복마다 `runs/<id>/iterations/iter-NNN/review.md`로 쌓이고, UI의 "산출물 보기"에서
리뷰 리포트(마크다운 렌더) · 개선본 · 원본 대비 diff를 탭으로 볼 수 있습니다.

### 방법 B — CLI로 바로 실행 (+ 대시보드 동시 기동)
```powershell
# 로컬 문서
npm run review -- "samples/sample.md" --open

# 웹사이트
npm run review -- "https://example.com"

# 로컬 코드 (파일 또는 폴더)
npm run review -- "C:/path/to/project/src"
```
실행하면 대시보드가 함께 떠서(기본 http://localhost:4517) 진행 상황을 실시간으로 보여주고,
루프가 끝나도 대시보드는 계속 유지됩니다(Ctrl+C로 종료).

### 대시보드만 실행
```powershell
npm run dashboard -- --open
```

### 런 목록 보기
```powershell
npm run list
```

## 주요 옵션 (`review`)
| 옵션 | 설명 | 기본 |
| --- | --- | --- |
| `-k, --kind` | 입력 종류 강제 (`document`/`url`/`code`) | 자동 감지 |
| `-p, --port` | 대시보드 포트 | 4517 |
| `--patience <n>` | 비개선 연속 허용 횟수 → 초과 시 중단 | 2 |
| `--min-delta <n>` | 개선으로 인정할 최소 점수 상승폭 | 0.5 |
| `--max-iter <n>` | 안전 상한 반복 수(폭주 방지) | 40 |
| `--model <m>` | claude 모델 (`opus`/`sonnet` 등) | CLI 기본 |

> 대시보드의 고급 옵션에는 **동일문제 반복감지(`recurWindow`, 기본 3)** 도 있습니다.
| `--no-dashboard` | 대시보드 없이 실행(끝나면 종료) | — |
| `--open` | 브라우저 자동 열기 | — |

## 동작 방식 — 점진적(incremental) 누적 리뷰
매 회차 전체를 처음부터 다시 검사하지 않습니다.
- **1회차**: 대상을 검토해 발견한 **모든 지적사항을 대장(`findings.md`)에 누적**합니다.
- **2회차 이후**: 대장의 **열린 항목을 해결**하고, **이전에 놓친 새 문제만** 추가합니다(전체 재검사 X).
- 에이전트는 매 회차 전체 이슈가 아니라 **델타(해결한 항목 번호 + 새로 발견한 항목)만** 보고하므로 토큰을 아낍니다.

## 종료 규칙
- **열린 항목이 모두 해결되고 새로 발견된 문제도 없음** → 완료
- `done=true` (에이전트가 마무리 선언) → 완료
- 같은 항목이 `recurWindow`회 연속 안 풀림 → **자동 일시정지**(검토 요청 알림)
- `max-iter` 도달 → 안전 중단

다음 회차 입력은 항상 **직전 회차 개선본(`current/`)** 을 이어받고, 점수가 가장 높았던 스냅샷은 `runs/<id>/best/`에 따로 보존됩니다.

## 일시정지 / 중단 (토큰 절약)
대시보드 상세 화면의 **⏸ 일시정지 / ▶ 재개 / ⏹ 중단** 버튼으로 런을 직접 제어합니다.
일시정지/중단은 **다음 `claude` 호출 직전에 적용**되므로(진행 중인 반복은 끝까지 마침),
일시정지 상태에서는 **토큰을 전혀 사용하지 않습니다**.

## 동일 문제 반복 자동 일시정지 + 알림
QA/검수 이후에도 **같은 문제(`remaining_issues`)가 최근 `recurWindow`(기본 3)회 연속** 등장하면,
루프가 **스스로 일시정지**하고 다음을 수행합니다:
- 상세 화면 상단에 **강조 배너**로 반복된 문제를 표시
- 토스트 + (허용 시) **데스크톱 알림** + 비프음으로 다음 메시지를 띄움:
  > "불필요한 토큰 사용을 막기 위해 일시정지 됐습니다. 기존 내용 확인 부탁드립니다."

헤더의 **🔔 알림** 버튼으로 데스크톱 알림을 켤 수 있습니다. 내용 확인 후 **재개** 또는 **중단**을 선택하면 됩니다.

## 산출물 구조
```
runs/<런ID>/
  input.<ext>            원본 스냅샷
  input.meta.json        입력 메타
  findings.md            지적사항 대장(누적 체크리스트 — 열린/해결 항목)
  state.md               압축된 진행 상태
  run.json               전체 상태/점수/대장/로그 (대시보드 데이터 소스)
  iterations/iter-NNN/
    review.md            그 회차 작업 노트
    improved.<ext>       그 회차 개선본
    score.json           점수·차원·근거·해결/새발견·열린항목
  current/               직전 회차 개선본(다음 회차 입력)
  best/                  점수 최고 스냅샷의 review.md + improved.<ext>
```

## 구조
- `src/cli.ts` — CLI (commander)
- `src/input/adapters.ts` — 문서/URL/코드 입력 어댑터
- `src/orchestrator/` — `runner`(루프) · `claudeAgent`(헤드리스 호출/JSON 파싱) · `prompts`(루브릭) · `state`(영속화)
- `src/dashboard/` — Express + SSE 서버 및 정적 프론트(외부 CDN 의존 없음)
