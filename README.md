# auto-dr — 자율 리뷰·개선 루프

로컬 문서 / 웹사이트 URL / 로컬 코드를 입력하면, **사람 개입 없이** AI가 스스로

1. 문제점·개선 추천점을 찾아 **지적사항 대장**에 쌓고,
2. 그 지적을 반영한 **개선본**을 만들고,
3. 그 개선본을 **별도의 독립 채점기(루브릭 기반)** 로 채점해서,
4. 점수가 오르면 계속, 떨어지거나 정체되면 멈추고 **베스트 버전을 보존**합니다.

완료 직전에는 **가장 강한 모델로 전체를 한 번 더 정밀검증**하고, 비용·반복은 상한으로 보호됩니다.

매 반복은 새로운 헤드리스 호출이며, 직전 전체 대화 대신 **압축된 `state.md` + 현재 베스트 산출물 + 열린 지적사항**만 넘겨 토큰을 아끼면서 작업을 이어갑니다. 산출물은 전부 로컬 `runs/`에 저장되고, 진행 상황은 실시간 로컬 웹 대시보드로 모두 볼 수 있습니다.

기본 백엔드는 **Claude Code CLI(`claude -p`, 구독 사용)** 이며, 원하면 **Anthropic / OpenAI / Gemini / Together AI / NVIDIA Nemotron API 키**를 넣어 직접 호출할 수도 있습니다.

## 요구사항
- Node.js 18+ (검증: v22)
- 기본 백엔드(`cli`)를 쓰려면 `claude` CLI 로그인 완료 (Claude Code)
- (선택) API 직접 호출을 쓰려면 해당 provider의 API 키
- (선택) URL의 JS 렌더링(SPA)을 쓰려면 Chromium: `npx playwright install chromium`

## 사용법

### 방법 A — 대시보드에서 직접 (권장, CLI 불필요)
```powershell
npm run dashboard -- --open
```
브라우저에서 왼쪽 **"➕ 새 리뷰 시작"** 패널로:
1. `파일 업로드`(드래그앤드롭/클릭) 또는 `URL / 경로` 입력
2. **리뷰 초점** 선택 (종합 / 보안 / 가독성 / 성능 / 문서 명료성 / SEO·마케팅 등)
3. (선택) 고급 옵션에서 종류·모델·비용 상한·patience·최대 반복·평가기준 등 지정
4. **리뷰 시작** → 백그라운드로 자율 루프가 돌고, 진행·점수·비용·토큰·리뷰 결과가 실시간으로 표시됩니다.

작업 노트(마크다운 렌더) · 개선본 · 원본 대비 diff · 변경 내역을 탭으로 볼 수 있습니다.

## 동작 방식 — 점진적(incremental) 누적 리뷰
매 회차 전체를 처음부터 다시 검사하지 않습니다.
- **1회차**: 대상을 전체 검토해 발견한 **모든 지적사항을 대장(`findings.md`)에 누적**하고, 개선본 전체를 작성합니다.
  - 입력이 매우 크면(24,000자 초과) **섹션 경계로 분할(map-reduce)** 해 파트별로 개선하고 이어붙입니다 → 컨텍스트 초과·본문 누락 방지.
- **2회차 이후**: 대장의 **열린 항목을 해결**하고, **이전에 놓친 새 문제만** 추가합니다(전체 재검사 X).
  - 개선본 전체를 다시 출력하지 않고 **바뀌는 부분만 `edits`(부분 수정 패치)** 로 받아 적용 → 출력 토큰 절약.
  - `find`가 **본문에 유일할 때만** 치환(다중 일치는 건너뜀)해 오적용을 막고, **실제로 수정된 항목만** 해결 처리합니다(가짜 해결 방지).
  - 큰 문서는 열린 지적과 **관련된 섹션만 발췌**해 입력으로 보냅니다 → 입력 토큰 절약.
- 새 지적은 정규화 일치뿐 아니라 **유사도 기반으로 근접 중복까지 병합**합니다(표현만 다른 같은 지적 제거).
- 점수가 베스트 이상이면 채택(대장 갱신), 미만이면 그 회차는 폐기하고 **항상 베스트에서 재시도**합니다.
  - 단, 개선본이 베스트 대비 **절반 미만으로 축소**되면(본문 누락 의심) 점수와 무관하게 채택을 거부합니다(콘텐츠 보존 가드).

## 자동 모델 라우팅 + 자동 승급
모델을 지정하지 않으면(`auto`, 기본) 입력 분량·종류·초점을 보고 **모델 사다리**를 정합니다(LLM 호출 없는 코드 휴리스틱 → 비용 0).
- 짧은 코드·심층(보안/정합성/버그/성능) 초점 → **opus 직행**
- 중간 분량 → **sonnet → opus**
- 대용량(2만 자 초과) → **haiku → sonnet → opus** 계단식

진행이 막히거나(같은 지적 반복), 개선이 정체되거나, 한 모델에서 `maxAttemptsPerModel`회를 소진하면 **더 강한 모델로 한 단계 자동 승급**해 재시도합니다. 모델을 직접 지정하면 그 모델로 고정되며 승급은 일어나지 않습니다.

> **예산 인지 승급**: 잔여 예산이 상위 모델 1콜 추정비보다 적으면 승급하지 않습니다(비싼 모델로 올라갔다가 비용 상한에 막혀 실패하는 상황 방지). 비용 상한으로 호출이 거부되면 크래시가 아니라 `stopped_cost`로 깔끔히 마칩니다.

## 비용·반복 상한 (폭주 방지)
- `maxTotalCostUsd` (기본 **$2.00**): 누적 비용이 한도를 넘으면 다음 호출 전에 종료(`stopped_cost`). 대시보드에서 변경·해제 가능.
- `maxCostPerIterUsd`: 회차당 비용 상한(선택). 실제 호출에는 회차 상한과 **총상한 잔여분 중 작은 값**이 적용됩니다.
- `maxIterations` (기본 8): 안전 상한 반복 수. 실제 종료는 보통 완료/정체/비용상한이 먼저 겁니다.

> 런이 **수렴 전에 끊긴다면** 대개 위 상한에 막힌 것입니다. 더 높은 품질을 원하면 대시보드 고급 옵션에서 `maxTotalCostUsd`·`maxIterations`를 올리세요.

## 종료 규칙
- **열린 항목이 모두 해결되고 새로 발견된 문제도 없음** → 완료
- (맞춤 평가기준 사용 시) 종합 점수가 **합격선(`passThreshold`) 도달** → 완료
- `done=true` (에이전트가 마무리 선언) → 완료
- `patience`회 연속 개선 정체/점수 하락 → 자동 종료(토큰 절약)
- 같은 항목이 `recurWindow`회 연속 안 풀림 → 더 강한 모델이 남았으면 승급, 없으면 **자동 일시정지**(검토 요청 알림)
- 총비용 상한 도달 → 종료 / `max-iter` 도달 → 안전 중단

## 일시정지 / 중단 (토큰 절약)
대시보드 상세 화면의 **⏸ 일시정지 / ▶ 재개 / ⏹ 중단** 버튼으로 런을 직접 제어합니다.
일시정지/중단은 **다음 호출 직전에 적용**되므로(진행 중인 반복은 끝까지 마침),
일시정지 상태에서는 **토큰을 전혀 사용하지 않습니다**. 중단은 진행 중인 호출까지 즉시 abort 합니다.

## 동일 문제 반복 자동 일시정지 + 알림
검토 이후에도 **같은 문제가 최근 `recurWindow`(기본 3)회 연속** 등장하고 **더 승급할 모델이 없으면**,
루프가 **스스로 일시정지**하고 다음을 수행합니다:
- 상세 화면 상단에 **강조 배너**로 반복된 문제를 표시
- 토스트 + (허용 시) **데스크톱 알림** + 비프음으로 다음 메시지를 띄움:
  > "불필요한 토큰 사용을 막기 위해 일시정지 됐습니다. 기존 내용 확인 부탁드립니다."

헤더의 **🔔 알림** 버튼으로 데스크톱 알림을 켤 수 있습니다. 내용 확인 후 **재개** 또는 **중단**을 선택합니다.

## 고급 기능

### 독립 채점기 (기본 켜짐)
개선한 모델이 **자기 점수를 매기면** 점수가 부풀려져 잘못된 시점에 "완료"로 멈출 수 있습니다. 그래서 매 회차 **최종 개선본을 별도 모델(`judgeModel`, 기본 `sonnet`)이 독립적으로 채점**합니다. 채택/합격선 판정은 이 점수를 사용합니다. 끄려면 `useJudge:false`, 채점기 모델은 `judgeModel`로 지정. (산출물이 매우 크면 채점 호출의 컨텍스트 보호를 위해 자가 점수로 자동 폴백)

### 마무리 정밀검증 패스 (기본 켜짐)
완료를 선언하기 직전에 **가장 강한 모델로 전체를 한 번 더 꼼꼼히** 점검합니다(최대 1회). 점진 수정이 놓친 일관성·표기·누락을 마감 품질로 끌어올립니다. 회차·예산 여유가 있을 때만 끼워 넣습니다. `finalPass:false`로 끄거나 `finalPassModel`로 모델 지정.

### 후속 패스 깊이 조절 (품질↔토큰)
2회차 이후는 토큰을 아끼려 기본적으로 렌즈 1개·검증 생략입니다. 품질을 더 원하면 `laterPassLenses`(렌즈 수)·`laterPassVerify`(검증 on)를 올리세요.

### 해결 정합성 & 후반 집중 (R1·R2)
- **해결은 실제 수정으로만 인정**: 2회차 이후 "해결"은 그 항목을 고친 `edit`(`findingIds` 태그)이 있을 때만 닫힙니다. "말로만 해결"은 무시되어 열린 채로 남습니다.
- **후반 새 발견 억제**: 2회차 이후 신규 지적은 회차당 최대 `laterPassMaxNew`(기본 3)개·중요도 우선으로만 받아, **열린 항목 해결을 우선**합니다(대장 폭증 방지). 1회차·마무리 패스는 제한 없음.

### 다각도 리뷰 패널 (panel 모드)
한 번의 호출 대신 **여러 렌즈(정합성·구조·표현·사실성 등)로 병렬 리뷰 → 중복 제거 → 진위 검증 → 에디터가 반영**하는 다단계 패스입니다. 렌즈 리뷰·검증은 저렴/빠른 모델(Claude는 haiku, **Nemotron은 추론 off 경량 패스**)로, 실제 개선만 라우팅 모델로 수행해 품질을 높이면서 비용·시간을 관리합니다. (2회차 이후는 렌즈 1개·검증 생략의 경량 패스)

### 대시보드 진행 시각화 & 실시간 UX
긴 리뷰가 "멈춘 것처럼" 보이지 않게, 진행 중 화면을 실시간으로 보여줍니다:
- **실시간 생성 뷰(스트리밍)**: 모델이 토큰을 쓰는 즉시 working 화면의 "✍️ 실시간 생성" 박스에 흘러나옵니다(추론 과정 + 결과). SSE `live` 이벤트로 전달.
- **단계 추적 + 경과/예상 시간**: `리뷰 → 개선 → 채점` 단계를 펄스로 표시하고, 경과 시간(초 단위 카운트업)과 회차 예상 시간(회차당 평균 기반)을 함께 보여줍니다.
- **단계별 소요 간트**: 회차마다 리뷰/검증/개선/채점이 각각 얼마나 걸렸는지 **누적 막대(폭=소요시간 비례)** 로 시각화 — 어디서 시간이 드는지 한눈에.
- **전/후 렌더 비교(svg·html)**: "원본과 비교" 탭에서 원본·개선본을 **실제 렌더로 나란히**(스크립트 차단 샌드박스) 보거나 텍스트 차이로 토글. "개선본" 탭도 svg/html은 렌더 미리보기 + 다운로드.
- **인라인 에러 + 재시도**: 런이 실패하면 원인을 **사람 말로 풀이**(예: 504=서버 일시 혼잡)하고, **[처음부터 다시 실행]**(베스트 없어도 동작) · **[베스트에서 이어하기]** 버튼을 그 자리에서 제공.

### 평가 관점 (채점 기준 원클릭)
문서·코드도 URL처럼 **"평가 관점" 셀렉트 하나로 채점 차원을 전환**합니다: `자동 추천`(입력 종류·초점에 맞춤) · `일반 문서` · `마케팅·카피` · `이력서` · `기술·코드` · `직접 설정`. 선택하면 해당 rubric 템플릿이 자동 적용되고, `자동 추천`은 입력 종류(코드→기술·코드)·초점(이력서/마케팅 키워드)으로 추천합니다. (URL은 이 자리 대신 "리뷰 범위"가 차원을 전환)

### 맞춤 평가기준 (rubric)
대시보드에서 **템플릿(문서/코드/이력서/마케팅)** 을 골라 차원·가중치·합격선을 편집합니다(평가 관점에서 `직접 설정` 선택 시). 종합 점수는 각 차원 점수의 **가중평균**으로 시스템이 다시 계산하고, 합격선에 도달하면 완료 처리합니다.

### 참고자료 첨부 (references)
"반드시 지켜야 할 기준" 문서를 함께 올리면, 시작 시 **체크리스트로 1회 요약**(`refs/`)해 **매 패스에 주입**합니다. 기준을 위반하는 부분은 지적·교정 대상이 됩니다.

### 변경 내역 (emitChanges)
켜면 매 부분 수정에 **이유·관련 지적 번호**를 함께 받아 `best/changes.md`로 누적하고, 원본이 오피스 문서면 `changes.<형식>`으로도 내보냅니다.

### A/B 비교
같은 입력을 **두 설정(A/B)** 으로 순차 실행(`/api/compare`, 같은 groupId)해 결과를 나란히 비교합니다.

### 멀티 프로바이더 (Anthropic / OpenAI / Gemini / Together AI / NVIDIA Nemotron)
대시보드의 **"백엔드 · 모델 · API 키" 팝업**에서 호출 방식을 고릅니다: Claude Code 구독(기본) / Anthropic / OpenAI / Gemini / Together AI / **NVIDIA Nemotron**. API provider면 해당 키로 **직접 호출**(자동 라우팅·승급 없이 지정 모델 고정). 키는 `data/providers.json`에 **로컬로만** 저장되며 `run.json`/설정엔 절대 안 들어가고, API로는 "어떤 키가 설정됐는지"만 노출합니다.
- **키 초기화**: 팝업에서 provider별 "초기화" 또는 "전체 초기화"로 저장된 키를 지웁니다(`DELETE /api/keys?provider=…`).
- **모델 id**: API provider는 모델 칸을 비우면 각 provider 기본 모델로 폴백합니다(`auto`는 Claude 자동 라우팅 전용 키워드라 API에는 전달하지 않음). Nemotron 기본 모델은 `nvidia/nemotron-3-super-120b-a12b`.

#### NVIDIA Nemotron 특이사항 (추론 모델)
Nemotron은 **추론(reasoning) 모델**이라 일반 채팅 모델과 다르게 처리됩니다(`integrate.api.nvidia.com`, OpenAI 호환):
- **깊은 리뷰**: 에디터·채점은 `detailed thinking on` + 큰 출력(`max_tokens` 65536)으로 사고를 충분히 돌려 지적·개선 품질을 확보합니다. (추론을 끄면 회차당 지적이 1개로 얕아짐)
- **렌즈 경량화(P2)**: panel 모드의 렌즈·검증은 `detailed thinking off`의 **빠른 스카우트 패스**로 처리(회차당 ~수 분 절약). 깊은 작업은 에디터가 담당. Claude가 렌즈에 haiku를 쓰는 설계와 동일한 취지.
- **스트리밍(P1)**: 응답을 **토큰 단위로 스트리밍**해 진행 중 화면에 실시간 표시(아래 "실시간 생성 뷰").
- **복원력**: 게이트웨이 5xx(특히 504)·네트워크 오류는 **백오프 후 자동 재시도**(4s→10s, 최대 3회). 응답이 토큰 한도에서 잘리면 조용히 빈 결과로 넘어가지 않고 명확히 오류로 알립니다. 추론 모델은 느려 호출 타임아웃은 12분으로 둡니다.

### 웹사이트(URL) 리뷰 — 구조·SEO·접근성 + 실제 코드
URL 입력은 보이는 텍스트만 보지 않고 **페이지를 구조적으로 분석**합니다:
- **SEO/메타**: title·description 길이, canonical·viewport·robots·lang, Open Graph/Twitter 카드, JSON-LD 구조화 데이터
- **구조/접근성**: heading 위계(h1~h6), alt 없는 이미지, 모호한 링크 텍스트("여기/클릭"), 빈 링크
- **실제 코드**: HTML 소스(발췌) + **같은 도메인 CSS/JS 파일**(개수·용량 제한)을 함께 묶어 마크업·스타일·스크립트 품질까지 리뷰
- **리소스/무게**, **SPA(클라이언트 렌더링) 감지 힌트** 포함
- **JS 렌더링(SPA 대응)**: `urlRender`로 `auto`(SPA 감지 시 헤드리스 브라우저로 렌더)·`on`(항상)·`off`. Playwright/Chromium 미설치거나 실패하면 정적 HTML로 자동 폴백(`npx playwright install chromium`).
- **다중 페이지 크롤**(`urlCrawl`, 최대 8쪽): 같은 도메인 내부 링크를 따라가 페이지별 구조를 요약.
- **링크 유효성 검사**(`urlCheckLinks`): 깨진 링크(4xx/5xx/연결오류) 탐지. 내부/사설 주소는 안전상 건너뜀.

리뷰 범위는 `urlMode`로 선택: `full`(기본, 종합) · `content`(본문 텍스트만) · `source`(코드 위주). **모드에 따라 평가 차원도 자동 전환**됩니다 — `content`=SEO·설득력 중심, `source`=마크업·접근성·성능·코드품질, `full`=하이브리드. (맞춤 평가기준을 주면 항상 그게 우선.) 모든 리소스 fetch는 **동일 출처만**, 크기·개수·타임아웃 제한 + SSRF 가드.

### 오피스 문서 입·출력
- **입력**: `.pdf`(pdf-parse) · `.docx`(mammoth) · `.pptx`(jszip) 텍스트 추출.
- **출력(원본 양식 유지)**: docx/pptx는 **업로드한 원본 파일을 그대로 두고 텍스트 내용만 제자리 교체**합니다 — 표·이미지·글꼴·레이아웃 보존(`editOfficeInPlace`). 문단 순서로 매핑하며, 구조가 크게 달라지면 안전하게 **깔끔한 새 문서로 재생성**(폴백). 제자리 수정이 되면 참고용 "정리본"(재생성)도 함께 제공. pdf는 재생성만.
- **왕복 검증**: docx/pptx 재생성 직후 다시 텍스트를 추출해 본문 손실이 없는지 점검하고, 의심되면 경고를 남깁니다.

## 산출물 구조
```
runs/<런ID>/
  input.<ext>            원본 스냅샷
  input.meta.json        입력 메타
  findings.md            지적사항 대장(누적 체크리스트 — 열린/해결 항목)
  state.md               압축된 진행 상태(다음 호출에 주입)
  run.json               전체 상태/점수/대장/비용/토큰 + 최근 로그 (대시보드 데이터 소스)
  log.jsonl              전체 로그 이력(append; run.json 에는 최근 200개만)
  refs/                  첨부 참고자료 추출본(ref-N.md) — 첨부 시
  iterations/iter-NNN/
    review.md            그 회차 작업 노트
    improved.<ext>       그 회차 개선본
    score.json           점수·차원·근거·해결/새발견·열린항목
  current/               가장 최근 개선본
  best/                  점수 최고 스냅샷: review.md + improved.<ext>
    improved.<docx|pptx|pdf>   원본 형식 재생성본(오피스 입력 시)
    changes.md / changes.<fmt> 변경 내역(emitChanges 사용 시)
```

## 구조
- `src/cli.ts` — CLI (commander): `review` / `dashboard` / `list`
- `src/config.ts` — 기본 설정·경로 상수(`DEFAULT_CONFIG`)
- `src/types.ts` — 공용 타입(`RunConfig`/`RunState`/`IterationResult` 등)
- `src/input/adapters.ts` — 문서/URL/코드 입력 어댑터 + 오피스 텍스트 추출 (`.svg`·`.xml`은 마크업으로 인식해 개선본을 같은 확장자로 저장)
- `src/output/export.ts` — 개선본/변경내역을 오피스 형식(docx/pptx/pdf)으로 재생성
- `src/orchestrator/`
  - `runner.ts` — 자율 반복 루프(채택·승급·종료·정합성/회귀 가드)
  - `claudeAgent.ts` — 헤드리스 호출/JSON 파싱(`runAgent`/`runPlain`)
  - `providers.ts` — Anthropic/OpenAI/Gemini/Together/**Nemotron** API 직접 호출 + 키 관리 + 비용 추정 (Nemotron은 스트리밍·추론 on/off·백오프 재시도)
  - `live.ts` — 진행 중 생성 토큰을 대시보드로 흘리는 인메모리 라이브 버스(SSE `live`)
  - `routing.ts` — 입력 트리아지 → 모델 사다리/자동 승급
  - `panel.ts` — 다각도 리뷰 패널(렌즈·검증·에디터) + 단계별 소요 측정
  - `judge.ts` — 독립 채점기(개선 모델과 분리된 모델이 점수 산정)
  - `chunked.ts` — 대용량 1회차 분할(map-reduce) 검토
  - `dedup.ts` — 지적사항 근접 중복 제거(문자 바이그램 유사도)
  - `limits.ts` — 전역 동시성 슬롯 + 세션 비용 상한
  - `prompts.ts` — 프롬프트/렌즈/섹션 스코프/청크 모드
  - `rubrics.ts` — 평가기준 템플릿·정규화·가중평균
  - `controls.ts` — 일시정지/재개/중단 신호 및 자식 프로세스 abort
  - `state.ts` — 런 영속화(run.json/findings.md/state.md/log.jsonl/산출물) + 이어하기/다시실행 시드
- `src/dashboard/` — Express + SSE 서버(변경 감지 `change` · 실시간 생성 `live`) 및 정적 프론트(외부 CDN 의존 없음)
